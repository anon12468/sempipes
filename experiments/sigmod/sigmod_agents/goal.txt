Entity resolution blocking on X2 (electronic products)

TASK
Write Python code that blocks duplicate candidates within X2.csv. Blocking must filter unlikely non-matches and output exactly 2,000,000 candidate pairs in output.csv (columns: lid (left instance id), rid (right instance id); left < right; no self-pairs; no duplicates).

- X2.csv = records to block
- Z2.csv (Y2) = ground truth for evaluation only. NEVER read or use Z2/Y2 when generating candidates
- sample_data/ is a small dev subset. Your algorithm MUST remain feasible on full X2 (~2 million records)
- Ground-truth pairs are transitively closed, stored once as (id1, id2) with id1 < id2
- Beat sample_data/blocking.py on recall where possible

CRITICAL: FEASIBILITY (solutions that violate this will fail)
Full X2 has ~2 million records. Your code MUST finish in under 30 minutes on a single machine with normal RAM.

A solution is INFEASIBLE and WRONG if it:
- Uses itertools.combinations (or nested loops) to emit all pairs among records sharing one token/key
- Performs O(k^2) work for any blocking bucket of size k (e.g. pairing every id in an inverted-index posting list)
- Builds a set/list of all candidate pairs and only then truncates to 2,000,000 rows

Why: one frequent token matching 10,000 records creates ~50M pairs. At 2M-record scale this runs for hours or exhausts memory.

SUGGESTED TECHNIQUES
Feature extraction (normalization, regex, rules) and blocking (inverted index with safeguards, LSH, Jaccard, etc.).

FORBIDDEN
- All-pairs enumeration or all-pairs similarity over the dataset
- O(k^2) pairing inside any blocking bucket
- Using ground truth (Z2/Y2) during candidate generation

OUTPUT
Save output.csv with exactly 2,000,000 rows.

SUCCESS METRIC
Maximize recall = (true matches in output) / (matches in ground truth), subject to ALL feasibility constraints above. High recall on sample_data/ alone is not enough if the code would explode on ~2M records.
